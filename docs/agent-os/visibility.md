# visibility.md

Users need to see the state that drives decisions.

Visible surfaces:

- Target plan JSON for machines.
- Context report Markdown for humans and agents.
- Runner report for task, family, model, and cost summaries.
- Gate receipt for pass/fail state.
- Charts for quick inspection.
- Blocker lists that persist until closed.

The UI or CLI should never hide missing official scorer status behind a green-looking summary.
