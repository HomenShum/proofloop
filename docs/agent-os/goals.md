# goals.md

ProofLoop goals are proof contracts.

Examples:

- "Run the repo's existing test suite and block done if it fails."
- "Classify the app and produce benchmark-family recommendations."
- "Scaffold a live URL smoke test and include it in the runner plan."
- "Verify official benchmark score artifacts with the upstream scorer."

Goal status must be derived from receipts: passed, failed, no_gate, blocked, or not_configured.
