# interrupts.md

Interrupts are user steering events.

Examples:

- "Also test the live URL."
- "Keep it under $100."
- "Use free models first."
- "Do not use memory mode."
- "Publish to npm after CI."

ProofLoop should convert interrupts into target-plan changes, runner budget changes, permission blockers, or new receipts. It should not overwrite prior goals unless the user explicitly replaces them.
